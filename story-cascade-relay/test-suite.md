# Advanced Cascade Smoke and Context Tests

## Purpose

Push beyond `guru` and single-pass chunking. These tests measure whether the cascade can preserve intent while splitting large context down from Agent 1's 1M window to Agent 3's 272K execution window.

## Test ladder

### Smoke A: Two-cycle story relay

**Goal:** Agents collectively write a short story over two full cascade cycles.

**Cycle 1:**

- Agent 1 loads story bible and global constraints.
- Agent 2 creates scene packets.
- Agent 3 drafts one scene seed per packet.
- Agent 2 integrates scene seeds into outline.
- Agent 1 verifies outline against global constraints.

**Cycle 2:**

- Agent 1 sends revision constraints and continuity findings.
- Agent 2 creates revision packets.
- Agent 3 revises each scene.
- Agent 2 integrates final story.
- Agent 1 verifies and returns final directive.

**Pass criteria:**

- `handoff.md` says `Cycle: 2/2` and `Status: complete`.
- outputs include cycle-1 outline, cycle-2 final story, continuity report.
- final directive is `USER:story-cascade-complete — 2 cycles verified.`
- at least 3 hidden facts from the story bible appear correctly in the final story.

### Smoke B: Needle-in-context continuity

**Goal:** Test whether details spread across the 1M context survive chunking.

Seed the story bible with 10 facts across distant sections:

1. protagonist cannot see the color blue;
2. city clocks run 7 minutes slow;
3. antagonist refuses to cross bridges;
4. sacred phrase is `lantern under snow`;
5. map has one intentionally false island;
6. sidekick has a silver allergy;
7. final door opens only to a question;
8. storm season lasts 19 days;
9. archive books are shelved by scent;
10. nobody may speak names at noon.

**Pass criteria:** final story uses at least 8/10 correctly and contradicts 0.

### Smoke C: Multi-cycle drift test

**Goal:** See how many full cascade cycles remain coherent.

Run cycles until either:

- 5 cycles complete, or
- continuity verifier finds a contradiction, or
- cumulative latency exceeds the run budget.

**Cycle pattern:**

1. outline
2. draft
3. revise for continuity
4. revise for emotional arc
5. final polish

**Pass criteria:** record max coherent cycle count.

### Smoke D: Compression budget test

**Goal:** Validate 1M -> 512K -> 272K compression.

- Agent 1 references a ~1M story bible.
- Agent 2 must produce packet manifests with estimated character counts.
- Agent 3 packets must stay below 220K chars to leave room for instructions and output.

**Pass criteria:** every Agent 3 packet file is below the configured limit and the final story still satisfies hidden-fact checks.

### Smoke E: Coding bridge analog

**Goal:** Transfer story relay mechanics back to coding/dev.

- Agent 1 provides a large repo context summary.
- Agent 2 creates implementation packets.
- Agent 3 writes one small fixture or doc output per packet.
- Agent 2 reviews and integrates.
- Agent 1 verifies.

**Pass criteria:** a verifier checks produced files and evidence packets.

## Metrics to capture

For every run:

- number of cycles completed;
- Agent 1 startup and turn latency;
- Agent 2 startup and per-packet latency;
- Agent 3 startup and per-packet latency;
- total wall time;
- packet count and size distribution;
- hidden facts preserved;
- contradictions found;
- final directive;
- verifier result.

## Recommended first implementation

Build only Smoke A first:

- 2 cycles;
- 6 scenes;
- synthetic ~1M story bible generated by script;
- Agent 3 packet target ≤220K chars;
- verifier checks handoff, outputs, cycle count, hidden facts, and packet sizes.

Do not attempt 5-cycle drift until Smoke A passes.
